# TaskManagePlus 온보딩 플로우 (최적화 버전)

TaskManagePlus는 **태스크 자동 생성 및 관리**와 **메모리뱅크(지식/회고/진행 로그) 시스템**이 통합된 프로젝트 관리 솔루션입니다.

온보딩 플로우는 아래와 같이 두 시스템이 자연스럽게 연동되도록 최적화되어 있습니다.

---

## 1단계: 기획서/요구사항 입력

- Notion, 파일, 링크, 외부 시스템 등 다양한 방식으로 프로젝트 기획서/요구사항을 입력합니다.
- (MCP 연동, 파일 첨부, URL 입력 등)

## 2단계: 태스크 자동 생성 및 파일명 규칙 선택

- 입력된 기획서를 바탕으로 주요 태스크가 자동 분할/추천됩니다.
- 온보딩 과정에서 태스크 파일명 규칙(주제명만/우선순위+주제명)을 선택합니다.
- 선택한 규칙에 따라 `tasks/` 폴더에 태스크가 개별 파일로 생성됩니다.
- 태스크 항목(필드)은 템플릿을 기준으로 하되, 언제든 자유롭게 커스터마이즈할 수 있습니다.

## 3단계: 태스크 진행/상태/회고 관리

- 생성된 태스크의 상태, 우선순위, 담당자, 진행 로그, 회고/피드백 등을 관리합니다.
- 소요시간에 영향을 준 주요 원인 등도 기록해 태스크 최적화에 활용합니다.

## 4단계: 메모리뱅크(Memory Bank) 설정 및 연동

- 태스크 관리와 동시에, 프로젝트의 주요 결정, 구조, 진행 상황, 회고, 학습 내용 등을 `memory-bank/` 폴더에 체계적으로 기록합니다.
- 회고/피드백, 진행 로그 등은 태스크와 메모리뱅크 양쪽에 연동/참조될 수 있습니다.
- 대규모 프로젝트는 도메인별로 하위 디렉토리 분리도 가능합니다.

### [메모리뱅크 온보딩 안내]

- memory-bank 폴더에는 아래와 같은 파일이 기본 제공됩니다:
  - `00-project-overview.md`: 프로젝트 목표/범위/주요 구조/시작 가이드
  - `01-architecture.md`: 시스템 아키텍처, 설계 패턴, 데이터 흐름, 기술 결정 등
  - `02-components.md`: 주요 모듈/서비스/외부 API/확장 구조 등
  - `03-development-process.md`: 작업 흐름, 브랜치 전략, 코드 리뷰, 배포 등
  - `04-api-documentation.md`: 엔드포인트, 요청/응답 예시, 인증, 에러 코드 등
  - `05-progress-log.md`: 날짜별/버전별 주요 변경 사항, 구현 연대기, 회고 등
- 각 파일은 실제 프로젝트 상황에 맞게 자유롭게 확장/수정하세요.
- 예시: `05-progress-log.md`에는 태스크 회고/피드백, 주요 결정, 학습 내용 등을 타임라인 형태로 기록

#### [사용자가 온보딩에서 직접 지정/설정할 수 있는 항목]

- memory-bank의 구조(도메인별 하위 폴더, 추가 파일 등)
- 각 문서의 작성/업데이트 책임자
- 기록/갱신 주기(예: 주요 이벤트 발생 시, 주간/월간 등)
- 작성 방식 및 상세 수준(예시, 코드, 다이어그램 등)
- 진행 로그/회고 기록 방식(자동/수동, 요약/원문 등)
- 공유/접근 권한(협업/외부 공개 등)
- 템플릿/포맷 커스터마이즈(항목 추가/삭제/수정)
- 위 항목들은 온보딩 단계에서 자유롭게 설정할 수 있으며, 운영 중에도 언제든 변경 가능합니다.

### [태스크와 메모리뱅크 연동 구조 및 예시]

- TaskManagePlus에서는 태스크(Task)와 메모리뱅크(Memory Bank)가 유기적으로 연동되어 프로젝트의 실행과 기록, 회고가 자연스럽게 이어집니다.

#### 연동 흐름 예시

1. **태스크 생성/진행**
   - 요구사항/기획서를 바탕으로 태스크가 자동 생성되고, 진행/상태/회고/피드백이 기록됩니다.
2. **태스크 회고/피드백 작성**
   - 각 태스크 완료 시, 회고/피드백(실제 소요 시간, 난이도, 느낀 점, 소요시간 영향 요인 등)을 기록합니다.
3. **메모리뱅크로 연동**
   - 태스크의 회고/피드백, 주요 결정, 학습 내용 등을 `memory-bank/05-progress-log.md`에 요약/참조/복사하여 타임라인 형태로 기록합니다.
   - 필요시 태스크 파일에서 memory-bank의 관련 문서(아키텍처, 컴포넌트, API 등)로 직접 링크/참조할 수 있습니다.
4. **통합 회고/분석**
   - memory-bank의 진행 로그, 구조/결정/회고 기록과 태스크 데이터를 함께 분석하여 프로젝트의 개선점, 주요 인사이트, 통계 리포트 등을 도출할 수 있습니다.

#### 실제 예시

- 태스크 파일의 회고/피드백 섹션에 작성한 내용을 아래와 같이 memory-bank/05-progress-log.md에 연동:

```
## 2025-06-01 | 회원가입/로그인 기능 구현 완료
- 실제 소요 시간: 7일
- 주요 원인: 소셜 로그인 연동 이슈, OAuth 문서 부족
- 느낀 점: 인증 플로우 표준화의 중요성 인식
- 개선점: 인증 모듈 템플릿화, 문서화 강화 필요
```

- 태스크에서 "관련 문서" 항목에 memory-bank/01-architecture.md 등으로 직접 링크

## 5단계: 통합 관리 및 분석

- 태스크와 메모리뱅크의 데이터를 바탕으로 프로젝트 진행률, 회고, 통계, 개선점 등을 통합적으로 분석/관리합니다.
- 예시: 태스크별 소요시간, 주요 결정, 회고/피드백을 한눈에 볼 수 있는 리포트 작성

---

> TaskManagePlus는 태스크 자동 생성/관리와 메모리뱅크(지식/회고/진행 로그) 시스템이 유기적으로 연동되는 구조입니다. 온보딩 플로우를 따라가면 두 시스템을 자연스럽게 통합 운영할 수 있습니다.

## 태스크 파일 네이밍 규칙(온보딩에서 선택)

- 온보딩 시 아래 두 가지 파일명 규칙 중 하나를 선택할 수 있습니다.
- 언제든 운영 중에도 자유롭게 변경 가능합니다.

### 1. 기본 파일명 규칙

- `tasks/task-주제명.md`
- 예시: `tasks/task-로그인.md`, `tasks/task-회원가입.md`, `tasks/task-커뮤니티-CRUD.md`

### 2. 우선순위 포함 파일명 규칙

- `tasks/task-우선순위-주제명.md` (우선순위는 숫자)
- 예시: `tasks/task-1-로그인.md`, `tasks/task-2-회원가입.md`, `tasks/task-3-커뮤니티-CRUD.md`

> 온보딩 단계에서 아래와 같이 파일명 규칙을 선택하세요.
>
> **Q. 태스크 파일명을 어떤 방식으로 관리하시겠습니까?**
>
> 1. 주제명만 (예: task-로그인.md)
> 2. 우선순위+주제명 (예: task-1-로그인.md)
>
> 선택 후, 모든 신규 태스크는 해당 규칙을 따릅니다. 운영 중에도 언제든 변경 가능합니다.
>
> **다음 단계:** 파일명 규칙 선택이 끝나면, 곧바로 메모리 뱅크(Memory Bank) 설정 단계로 이어집니다. 프로젝트의 회고, 피드백, 주요 결정사항, 학습 내용 등을 효과적으로 기록·분석할 수 있도록 메모리 뱅크를 꼭 설정하세요.
>
> **Q. 태스크매니저 온보딩을 마쳤습니다. 바로 이어서 메모리뱅크 온보딩을 실행하시겠습니까?**
>
> 1. 예, 바로 메모리뱅크 온보딩을 시작합니다.
> 2. 아니오, 나중에 따로 진행하겠습니다.

## 기존 태스크 파일 일괄 변경 자동화

- 이미 생성된 태스크 파일이 있다면, 아래 스크립트로 파일명을 일괄 변경할 수 있습니다.
- 예시(Bash):

```bash
# 우선순위가 태스크 파일 내에 "- 우선순위: N" 형식으로 있을 때
cd tasks
for file in task-*.md; do
  prio=$(grep "- 우선순위:" "$file" | awk '{print $3}')
  base=$(echo "$file" | sed 's/^task-//;s/\.md$//')
  mv "$file" "task-$prio-$base.md"
done
```

- 반대로, 우선순위 없는 파일명으로 되돌릴 때는 파일명에서 숫자만 제거하면 됩니다.

> 자동화 스크립트는 프로젝트 상황에 맞게 커스터마이즈해 사용하세요.

## 태스크 생성/관리 규칙

- 모든 태스크는 `tasks/` 폴더에 **개별 파일**(`task-*.md`)로 생성/관리합니다.
- 태스크 파일명은 위에서 선택한 규칙에 따라 작성합니다.
- 템플릿은 `templates/` 폴더에 위치하며, 신규 태스크 생성 시 복사해 사용합니다.
- 태스크 상태/진행/회고/피드백 등은 각 태스크 파일 내에서 관리합니다.
- **모든 날짜/시간 기록은 시스템의 현재 시간 또는 온라인에서 서울(Asia/Seoul) 기준 시각을 사용합니다.**

## 태스크 생성 방법

1. `templates/` 폴더에서 `task-template.md` 파일을 복사합니다.
2. `tasks/` 폴더에 선택한 파일명 규칙에 따라 파일명을 지정해 붙여넣습니다.
3. 태스크 제목, 설명, 우선순위, 상태, 담당자, 생성일 등 주요 항목을 작성합니다.
   - **날짜/시간은 시스템 시간 또는 온라인 서울 기준 시각을 참고해 기록합니다.**
4. 진행 로그(History), 회고/피드백(Feedback) 등은 태스크 진행에 따라 파일 내에 추가 기록합니다.

## 예시

```
tasks/
├── task-로그인.md
├── task-회원가입.md
├── task-커뮤니티-CRUD.md

또는

├── task-1-로그인.md
├── task-2-회원가입.md
├── task-3-커뮤니티-CRUD.md
```

## 참고

- 태스크 파일은 Git으로 버전 관리되며, 변경 이력/사유를 자동으로 추적할 수 있습니다.
- 태스크별로 세부 진행상태, 회고, 실제 소요 시간, 난이도 등을 기록해 프로젝트 회고/분석에 활용하세요.
- 템플릿은 자유롭게 커스터마이즈 가능합니다.
- **날짜/시간 기록 시 시스템 시간 또는 온라인 서울 기준 시각을 반드시 확인하세요.**

## 태스크 기록 항목 커스터마이징 안내

- 태스크에 기록해야 할 내용(항목, 형식 등)은 사용자가 자유롭게 커스터마이즈할 수 있습니다.
- 기본 제공 항목과 구조는 `templates/task-template.md` 파일을 기준으로 합니다.
- 필요에 따라 항목을 추가/삭제/수정하여 프로젝트에 맞게 활용하세요.
